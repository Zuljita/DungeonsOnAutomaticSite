User manual

Run a dungeon before the table gets restless.

This manual is the public guide for Dungeons on Automatic. It explains the workflow, what each control means, how to export useful files, and how to recover when a seed has developed professional objections.

Quick start

  1. Choose a dungeon size, biome mix, party CER, and challenge profile.
  2. Select the source books your table actually uses.
  3. Generate a seed. Review the GM map, player map, and room list.
  4. Lock any rooms you want to preserve, then reroll specific details as needed.
  5. Export the map and room key into your prep tool or VTT.
For a one-shot, start with a medium dungeon, standard generosity, and a moderate challenge profile. Raise danger after you know the party's appetite for terrible architectural decisions.

Maps and room keys

The GM map contains everything: room numbers, secret doors, hazards, keys, encounters, and notes. The player map removes hidden or GM-only information so it can be safely shared during play.

Room keys

Each room entry is meant to be used at the table. It should include room purpose, visible features, encounter pressure, treasure, hazards, doors, and tags that explain why the room belongs where it does.

Stage Manager

The Stage Manager is the in-character director layer. Kovath, Zarlzazz, Vorga, and Mizzik narrate what the generator is doing and provide dry operational commentary while the dungeon takes shape.

The jokes are flavor, but the status messages are still useful. If a generation pass fails, treat the message as a real diagnostic first and office misery second.

Troubleshooting

The dungeon feels too lethal

Lower challenge, reduce peril-heavy biomes, or reroll encounters while keeping the room layout locked.

The map feels too linear

Increase size, raise branch/dead-end tolerance, or add a second biome to force more internal transitions.

The export does not line up in a VTT

Confirm the target grid size, export scale, and whether the VTT expects pixels per square or square size in scene units.

I need to report a bug

Keep the seed, settings, browser, export format, and the room number or map area where the problem appears. That is the incident report. The staff will pretend to appreciate it.
