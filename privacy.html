Privacy and boundaries

Local-first dungeon prep.

Dungeons on Automatic is designed around local files, local settings, and optional local model providers. The public site hosts documentation and downloads; it is not a server database for your generated dungeons.

Local data

  • Generated dungeon state is saved as local JSON when you export or persist it.
  • Generation profiles and last-used settings are stored locally so your setup can survive a restart.
  • The app does not need a hosted account or server database to generate a dungeon.

Local LLMs

AI Narrative Polish is optional. When enabled, model calls are intended for local providers on your machine, such as Ollama, LM Studio, or llama.cpp. Browser builds may need local-provider CORS configuration before those calls are allowed.

The downloadable app does not bundle a local LLM runtime or model weights.

DFRPG content boundary

The bundled DFRPG data path uses structural data, tags, references, and original templates to support table prep. It is not a copy of copyrighted source-book text or stat blocks.

You should still use the source books you own at the table. Source-book filtering exists so the generator can stay aligned with your available library.
