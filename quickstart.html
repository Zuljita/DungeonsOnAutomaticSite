Your first dungeon in a minute

Generate, inspect, export, then go play.

Use this path for a first launch-week run. It favors stable defaults, player-safe exports, and a short final check before you bring the dungeon to the table.

Step 01

Choose safe defaults

Use Standard size, 4 biomes, Average challenge, 100% generosity, Standard door security, and the default required sources. Save the seed once the run looks usable.

Step 02

Read before rerolling

Open the GM map, player map, and room key. Check entrances, blocked paths, secret content, and the first few room entries before changing the seed.

Step 03

Export the right packet

Use player exports for handouts. Keep GM map images, Markdown room keys, placement TSV, Foundry scene JSON, and .dd2vtt files in your prep folder.

One-minute checklist

  • Confirm party CER and challenge profile match the table.
  • Confirm only source books your table uses are enabled.
  • Generate once with defaults before adding Dungeon Intent.
  • Save the seed and export the full JSON state.
  • Check the player map for hidden GM information.
  • Skim room keys for encounter and treasure pacing.
  • Import VTT files into a test scene before game time.
  • Keep the Known Issues section open during launch week.

Dungeons on Automatic is an unofficial free game aid. It expects you to use the source books you own and does not replace published GURPS or Dungeon Fantasy RPG rules.

Monster-library packages are public only after provenance and license review. Until a package is released, the monster page is a status and release surface rather than a stat-block catalog.
