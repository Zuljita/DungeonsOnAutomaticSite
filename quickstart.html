Start here

From install to room key.

Make one clean baseline before you tune anything. Generate with the defaults, read what DOA made, then export the version your table actually needs.

Step 01

Build a baseline

Open the Generate panel and click Generate Map. The defaults are enough for a first pass; set Party CER first if you already know it.

Step 02

Read before rerolling

Inspect Rooms & Contents, the adventure frame, and both map views. Check the entrance, obvious routes, encounters, hazards, treasure, and the first few room entries.

Step 03

Export the right packet

Give players a player-safe PNG or VTT file. Keep the GM map, Markdown room key, and Dungeon JSON with your prep.

One-minute checklist

  • Set Party CER and challenge for the current delvers.
  • Enable only the rulebooks and monster sources you own.
  • Generate once before adding a Session Brief.
  • Keep the seed and export the full Dungeon JSON.
  • Check the player map before you share it.
  • Skim the room key for encounter and treasure pacing.
  • Review the adventure frame and wandering procedures.
  • Import VTT files into a test scene before game time.

Give the directors something to work with

A Session Brief can name a theme, tone, set pieces, danger target, and character spotlight. Local AI can help interview you and turn those notes into structured suggestions.

Director authority is limited to typed tools. Source-filtered DFRPG data, deterministic generation, and validation approve every change before it enters the dungeon.
